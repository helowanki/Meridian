Features

Everything you need.
Nothing you don't.

Meridian ships a complete, curated toolchain for running Windows software on Apple Silicon. No configuration required — but everything is configurable.

🍷
Bottle Management
Create isolated Wine environments per game or app. Each bottle gets its own UUID-based prefix, icon, color, and settings. Export and import as .bottle.zip.
Core
⚡
Bundled Runtime
Wine, DXVK, vkd3d-proton, D3DMetal, GStreamer — all inside the .app bundle. Zero external installs. Works on a clean macOS system out of the box.
No setup
🎨
Graphics Backends
Switch between D3DMetal (Apple Metal native), DXVK (DX9/10/11), and vkd3d-proton (DX12) per bottle. DLL overrides and wineserver restart happen automatically.
Graphics
🚀
Launchpad Grid
Programs are shown as a 96 px icon grid — real icons extracted from .exe PE headers. Single click to select, double click to launch. Drag .exe to drop-install.
UI
📦
App Library
Built-in catalog of popular launchers, Windows runtimes, browsers, and tools. Downloads are cached locally — no re-download if you reinstall.
New
🔍
Program Scanner
Auto-scans drive_c/Program Files after installation. Filters system executables and helpers by size and name, then presents found apps for one-click import.
Automation
📊
Process Monitor
Live Wine process list refreshed every 2 seconds. Shows CPU%, RSS memory, and PID for each process. Kill individual processes or wipe all Wine processes at once.
Diagnostics
🎛️
Per-Bottle Inspector
A fixed right-panel inspector shows settings for the active bottle — Graphics API, Retina Mode, Sync Method, custom ENV vars, disk usage, and debug tools. Always visible, never a sheet.
UI
🔔
Menu Bar & Notifications
A status bar icon shows all your bottles and pinned programs. OS notifications fire when installs complete or bottles are ready. Works while the main window is closed.
macOS
🖥️
Retina Mode
Writes RetinaMode = Y to the Wine registry and restarts wineserver so the change takes effect immediately — no bottle restart needed.
Display
🔧
Environment Variables
Add, edit, and remove per-bottle KEY=value pairs that are merged into Wine's environment at launch time. Override any built-in variable without touching source code.
Advanced
🧪
DLSS / XeSS
Experimental upscaling hook via WINE_ALLOW_XR_UPSCALING. Results vary by game. Use with D3DMetal for best results on Apple Silicon.
Experimental

What's New

Release Notes

Every change to Meridian, documented in full.

1.0.0 2025 First public release
Added — Core
  • Native SwiftUI app targeting macOS 13+ on Apple Silicon with three-column NavigationSplitView layout
  • Complete Wine runtime bundled inside Meridian.app — no external install required. Includes Wine, DXVK, vkd3d-proton, D3DMetal, GStreamer, and supporting libraries
  • Accent color picker in Preferences — eight presets applied app-wide in real time via AccentColorManager
  • Onboarding wizard on first launch: runtime check, first bottle creation, and feature overview with animated steps
  • Home dashboard showing bottle count, program count, total launches, recent activity, and quick actions
Added — Bottles
  • Create, rename, export, and import Wine bottles. Per-bottle UUID prefix stored in ~/Library/Application Support/Meridian/Bottles/
  • Per-bottle icon picker: 32 quick-emoji, custom emoji field, and 8 color presets for the icon background
  • Automatic wineboot --init on creation with D3DMetal, msync, and Retina Mode applied as defaults
  • Bottle scanner: auto-detects installed programs in drive_c/Program Files, filters system executables by name and file size
  • Export bottles as .bottle.zip; import restores the prefix and registers the bottle immediately
  • Disk usage reported per bottle via async FileManager enumeration
Added — Graphics & Compatibility
  • D3DMetal backend via DYLD_LIBRARY_PATH pointing to lib/external/ and WINEDLLOVERRIDES
  • DXVK 2.x backend — DLLs copied into system32 / syswow64 on selection, removed automatically when switching away
  • vkd3d-proton backend with mirrored DLL management
  • msync and esync toggled via WINEMSYNC / WINEESYNC
  • Retina Mode written to HKCU\Software\Wine\Mac Driver via wine reg add and applied live
  • Experimental DLSS / XeSS hook via WINE_ALLOW_XR_UPSCALING
  • wineserver -k issued automatically on any graphics setting change
Added — Programs
  • Launchpad-style 96 px program grid with spring selection animation; single click selects, double click launches
  • Real icon extraction from Windows .exe PE resource section — Strategy A via Homebrew icoutils, Strategy B via native Swift PE parser
  • Icon cache at ~/Library/Application Support/Meridian/Icons/ keyed by exe path hash
  • Launch history with relative timestamps, stored per bottle, capped at 50 entries
  • Inline rename via double-click; remove from list without deleting the file on disk
  • Drag-and-drop .exe onto the grid to launch and register in one step
Added — App Library & Downloads
  • Built-in catalog across 8 categories: Launchers, Runtimes, Tools, Creative, Browsers, Social, Dev, Media
  • Installer cache at ~/Library/Application Support/Meridian/Installers/ — cached installers show "Open" instead of "Download"
  • Download sheet with circular progress ring, download speed (MB/s), ETA, linear progress bar, and live Wine install log
  • Cancel download or abort installation mid-way; wineserver -k cleans up all child installer processes
  • wineserver -w used in install flow to detect true completion including child installer windows
Added — macOS Integration
  • Menu bar status item with per-bottle program submenus, Kill Wineserver, Kill All, and open main window
  • OS notifications for bottle ready and install complete / failed via UserNotifications
  • Process Monitor — live Wine process list, 2-second refresh, with per-process CPU%, RSS, and kill
  • Export / import via NSSavePanel / NSOpenPanel with zip and unzip
  • Per-bottle custom environment variables added inline (KEY = value) and merged at launch time
Fixed
  • Pipe() without a reader blocked Wine when output exceeded the ~65 KB kernel buffer — replaced with /dev/null redirect for wineboot, wineserver -k, and reg add
  • Binding<Bottle> captured index at creation — array mutation after deletion caused Index out of range crash; fixed to search by id on every access
  • Bottle prefix path saved before removeAll — file deletion now uses the correct path even after the bottle is removed from the array
  • Context menu deletion left the prefix on disk — wineserver -k now runs synchronously before file removal with a one-second retry on failure
  • wineboot --init ran on every app launch — now skipped when drive_c already exists
  • Orphaned prefixes on disk (without a matching JSON entry) are now recovered automatically at startup
